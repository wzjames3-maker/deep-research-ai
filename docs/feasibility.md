# 可行性分析

## 技术可行性

| 维度 | 评估 | 风险 | 缓解措施 |
|---|---|---|---|
| LLM 对话与规划 | 依赖 OpenAI Chat Completion 协议，社区方案成熟（LangChain/LangGraph 等），可行性高 | 低 | 协议兼容性强，可切换多厂商 |
| MCP 集成 | MCP 协议规范明确，官方 SDK 可用（Python/TS），搜索源接入标准统一 | 低 | Phase 3.1 调研 MCP Server 生态 |
| Sub-agent 编排 | 主 Agent 拆分 3-5 个 Sub-agent，各跑 ≤2 轮循环——任务规模可控，Worker/Queue 模式即可 | 低 | 并发控制简单，无需复杂调度 |
| 实时进度推送 | 结构化工作台需实时展示 Sub-agent 执行状态——需 SSE/WebSocket | 中 | 选择成熟的实时通信库即可 |
| Web 前端 | 单页 Web 应用，结构化仪表盘，复杂度中等 | 低 | 社区组件库丰富 |
| 数据持久化 | 研究历史、用户数据持久化存储，数据模型不复杂 | 低 | PostgreSQL |

**结论：核心技术路径均有成熟方案支撑，无不可逾越的技术壁垒。**

## 资源可行性

| 维度 | 评估 |
|---|---|
| 人力 | 用户主导（Product Owner）+ AI Agent 辅助开发（全栈） |
| 开发周期 | 无硬性 Deadline，长期迭代演进 |
| 基础设施 | Docker 自托管，无额外基础设施依赖；需预置 LLM API Key |
| 运行时成本 | LLM API 调用费用（按量计费）+ 服务器/搜索 API 费用 |

**结论：无刚性资源约束，适合长周期渐进式开发。**

## 风险清单

| ID | 风险 | 概率 | 影响 | 缓解措施 |
|---|---|---|---|---|
| R-001 | LLM 生成的研究计划质量不稳定（子任务拆分不合理、遗漏关键角度） | 高 | 高 | ① 计划生成前多轮澄清交互（已设计） ② Prompt Engineering 迭代 ③ 可选：计划模板库 |
| R-002 | MCP Server 不可用/结果返回慢（外部依赖不可控） | 中 | 中 | ① 每个 Sub-agent 设置超时 ② 支持多 MCP Server 冗余 ③ 失败时降级输出部分结果 |
| R-003 | 长连接/流式推送在自托管环境下不稳定（Nginx 代理缓冲、连接中断） | 中 | 中 | ① Phase 2 内做 SSE POC 验证 ② 配置 Nginx 禁用缓冲 |
| R-004 | LLM API 速率限制导致 3-5 个并行 Sub-agent 无法同时执行 | 中 | 中 | ① 串行 fallback 模式 ② 指数退避重试 ③ 选高并发 tier 的 API plan |
| R-005 | 搜索结果质量依赖 MCP Server 提供方（可能出现低质/广告/死链） | 中 | 低 | ① 多源交叉验证 ② 结果来源标注，用户可溯源判断 |
| R-006 | Sub-agent 2 轮循环可能陷入搜索死循环（重复搜索同类页面） | 低 | 中 | ① 硬限制 2 轮 ② 已访问 URL 去重 ③ 信息增益阈值判断是否继续 |

**整体风险等级：中等可控。核心风险集中在 LLM 输出质量（R-001），其他风险均有常规工程手段应对。**

## POC 计划

## 初步技术方向（供 Phase 3.1 调研假设，非正式决策）

### 前端方向假设
- **假设**: React 18+ + TypeScript
- **理由**: 生态最丰富，结构化仪表盘组件库选择多，SSE 集成成熟
- **备选**: Next.js（如需 SSR）| Svelte（轻量）

### 后端方向假设
- **假设**: Python（FastAPI）
- **理由**: LLM 生态最强（LangChain/LangGraph 原生 Python），MCP SDK for Python 成熟，异步支持好
- **备选**: Node.js/TypeScript（前后端统一语言）| Go（高并发）

### AI 编排方向假设
- **假设**: LangGraph（StateGraph）或 自定义轻量编排
- **理由**: 天然适合多节点 DAG 编排（主 Agent → N 个 Sub-agent → 汇总），状态管理清晰
- **备选**: CrewAI | AutoGen | 纯手写编排

### 说明
以上假设会在 Phase 3.1 调研结束后基于调研结论重新评估，Phase 3.5 正式决策。
